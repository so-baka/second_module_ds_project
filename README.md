# Анализ лояльности пользователей Яндекс Афиши

## 1. Цели и задачи проекта

Команда маркетинга сервиса для заказа билетов на мероприятия **Яндекс Афиша** хочет лучше понимать поведение пользователей. В этом проекте проводится исследовательский анализ данных, чтобы определить, какие пользователи с большей вероятностью возвращаются на платформу и делают заказы повторно.

**Это позволит:**
- Быстро выявлять перспективных клиентов и предлагать им персонализированные условия
- Точно настраивать рекламу на аудитории с высокой вероятностью возврата
- Оптимизировать маркетинговые бюджеты
- Повысить общий уровень удержания клиентов

## 3. Описание данных

Выгрузка из базы данных SQL содержит следующие поля:

| Поле | Описание |
|------|----------|
| `user_id` | уникальный идентификатор пользователя |
| `device_type_canonical` | тип устройства (mobile/desktop) |
| `order_id` | уникальный идентификатор заказа |
| `order_dt` | дата создания заказа |
| `order_ts` | дата и время создания заказа |
| `currency_code` | валюта оплаты (rub/kzt) |
| `revenue` | выручка от заказа |
| `tickets_count` | количество купленных билетов |
| `days_since_prev` | дней с предыдущего заказа |
| `event_id` | идентификатор мероприятия |
| `service_name` | билетный оператор |
| `event_type_main` | тип мероприятия |
| `region_name` | регион проведения |
| `city_name` | город проведения |

## 4. Основные этапы работы

1. **Выгрузка данных** из PostgreSQL с помощью SQLAlchemy
2. **Предобработка данных**:
   - приведение выручки к единой валюте (рубль)
   - фильтрация выбросов по 99-му перцентилю
   - удаление отрицательных значений выручки
   - проверка и удаление дубликатов
   - создание профилей пользователей
3. **Исследовательский анализ данных (EDA)**:
   - сегментация пользователей по различным признакам
   - анализ возвратов в разрезе сегментов
   - проверка продуктовых гипотез
   - анализ влияния выручки и количества билетов
   - анализ временных характеристик
4. **Корреляционный анализ** с использованием `phik`
5. **Формулировка выводов и рекомендаций**

## 5. Ключевые результаты

| Показатель | Значение |
|------------|----------|
| Всего пользователей | 21 933 |
| Доля с 2+ заказами | 61.8% |
| Доля с 5+ заказами | 29.2% |
| Средняя выручка с заказа | 555.6 руб. |

**Основные факторы, влияющие на возврат пользователей:**
- Интервал между заказами (чем меньше, тем выше активность)
- Количество билетов в заказе (покупка 2–3 билетов — маркер лояльности)

## 6. Как запустить проект

### Требования
- Python 3.9+
- Jupyter Notebook
- PostgreSQL (для подключения к базе данных)

### Установка и запуск

bash
# 1. Клонировать репозиторий
git clone https://github.com/so-baka/second_module_ds_project.git
cd second_module_ds_project

# 2. Создать виртуальное окружение (рекомендуется)
python -m venv venv

# 3. Активировать окружение
# Для Windows:
venv\Scripts\activate
# Для Linux/Mac:
source venv/bin/activate

# 4. Установить зависимости
pip install -r requirements.txt

# 5. Создать файл .env на основе примера
cp .env.example .env
# Отредактировать .env, указав свои параметры подключения к БД

# 6. Запустить Jupyter Notebook
jupyter notebook
